# v0.11.2 - 2026-03-25

## 版本概览

`v0.11.2` 主要补的是图片查看体验。

在 `v0.11.1` 之后，模型输出图片已经可以稳定显示、稳定回放，但前端交互仍然偏“工程工具式”：

1. 点击聊天图片会进入统一的右侧抽屉
2. 下载入口只有一个，用户很难判断下载的是“当前预览图”还是“原图”

这套交互不符合豆包 / GPT / Gemini 这类大模型产品的常见心智模型。主流做法一般是：

1. 点击图片后进入居中的沉浸式大图查看器
2. 在查看器顶部提供清晰的操作按钮
3. 区分“当前图”和“原图”这两个概念

本次版本就是围绕这三个点做调整。

## 这次解决了什么问题

### 1. 预览交互改成居中图片查看器

旧方案是复用右侧 preview drawer。

这个方案的问题在于：

1. 它更适合“文件详情 / 文档引用 / OCR 内容”这类结构化信息浏览
2. 不适合模型生成图片这种以视觉沉浸感为主的内容
3. 用户点击图片后，预期是“放大看图”，而不是“进入文件详情侧栏”

所以这次改成了独立的图片查看器：

1. 点击聊天图片后，进入全屏遮罩式查看器
2. 图片居中展示，背景变暗
3. 顶部展示标题、来源和操作按钮
4. 不再打断当前聊天上下文，也不再挤占右侧信息面板

这套交互更接近 GPT / Gemini / 豆包的常见大图预览体验。

### 2. 下载动作不再语义模糊

之前预览里只有一个“下载图片”，但没有明确说明下载的是哪一份数据。

这次把下载语义拆清楚了：

#### 当前图

“当前图”指的是聊天界面当前稳定展示的那张图，也就是 `content_parts[].url` 对应的图片。

对于模型输出图片，它可能有两种来源：

1. 模型直接返回 `data:image/...`
2. 模型返回远程 URL，后端抓取后再转成稳定的 `data:` URL

现在“下载当前图”下载的就是这份当前稳定展示的数据。

重要的是：

- 这不是前端二次压缩后的图片
- 也不是浏览器截图
- 对于远程 URL 场景，它是后端抓取到的原始响应字节，再转成 `data:` URL 保存

也就是说，“下载当前图”拿到的是当前聊天里实际保存和展示的那份图像数据，不是前端压缩品。

#### 原图

“原图”指的是模型最初返回的原始地址。

本次新增了 `original_url` 概念：

1. 如果模型最初返回的是远程图片 URL，后端会把这个地址保存在 `original_url`
2. 前端图片查看器会展示“打开原图”按钮
3. 用户可以直接打开模型最初返回的原始地址

这解决了一个关键问题：

- 用户既可以下载当前会话中稳定保存的图片
- 也可以在需要时查看模型供应商返回的原始图资源

### 3. 文档图片和模型图片的职责边界更清晰

本次没有把所有预览都强行统一成一个组件，而是做了职责拆分：

#### 聊天消息里的模型图片

- 使用新的居中图片查看器
- 强调看图、放大、下载、打开原图

#### 文件列表 / 引用来源里的文档预览

- 继续使用右侧 preview drawer
- 适合展示文件元信息、OCR 文本、页码、引用片段

这样既保留了原有文件预览链路的优势，也让模型图片获得更符合心智的查看体验。

## 本次处理链路

### 模型图片查看链路

1. 后端 `/chat/ask` 返回 `content_parts`
2. 前端 `renderAssistantContent()` 渲染 `type=image`
3. 用户点击消息图片
4. 前端打开独立的图片查看器，而不是右侧抽屉
5. 查看器展示：
   - 当前图
   - 图片标题
   - 来源说明
   - 下载当前图按钮
   - 打开原图按钮（如果 `original_url` 存在）

### 当前图 / 原图数据链路

1. 模型返回图片
2. 后端 `LLMService` 提取图片内容
3. 如果图片是远程 URL：
   - 后端抓取图片字节
   - 转成稳定的 `data:` URL 写入 `url`
   - 同时把原始远程地址保存在 `original_url`
4. 如果图片本来就是 `data:` URL：
   - `url` 直接保存当前图
   - `original_url` 为空，因为没有额外的原始远程地址
5. 前端根据这两个字段决定展示：
   - “下载当前图”始终下载 `url`
   - “打开原图”仅在 `original_url` 存在时展示

## 关于“下载的是不是原图”

这里明确一下最终语义：

### 下载当前图

下载的是当前会话真正保存和展示的图片数据。

它不是浏览器压缩图，也不是前端重编码后的缩略图。

对于远程 URL 场景，它是后端抓到的原始字节内容，只是编码成了 `data:` URL 方便持久化和回放。

### 打开原图

如果模型最初返回了远程地址，则可以通过“打开原图”访问那份原始资源。

需要注意：

1. 这取决于模型供应商是否真的返回了单独的原图地址
2. 也取决于这个地址是否仍然有效

所以：

- “下载当前图”保证稳定、可回放
- “打开原图”用于尽可能访问供应商侧原始资源

这两者的职责是不同的。
