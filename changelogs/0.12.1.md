# v0.12.1 - 2026-03-26

## 版本概览

本版本主要修复 `/api/v1/chat/ask` 的一个兼容性问题：

1. 当用户选中图片附件发问时，部分 OpenAI-compatible 提供方会拒绝数组格式的 `messages[].content`。
2. 之前这类下游 400 会直接透传到 CaiBao，导致接口返回 `400 Bad Request`。
3. 现在会自动识别这类报错并降级为纯文本重试，避免聊天请求直接失败。

## 新增内容

1. 新增图片附件问答降级重试的 API 回归测试。
2. 新增版本记录，说明本次兼容性修复范围。

## 主要变更

1. 扩展 `LLMService` 的图片降级重试判定，兼容以下报错语义：
   - `expected a string`
   - `got an array`
   - `invalid type for 'messages[1].content'`
2. 当命中上述报错时，自动去掉图片数组输入，并改用附件 OCR/文本回退内容再次请求模型。
3. 保持原有支持视觉输入的模型行为不变，仅在下游明确拒绝该请求格式时才触发降级。

## 兼容性说明

1. 本次修复不改变 `/api/v1/chat/ask` 的请求结构和返回结构。
2. 仅影响“带图片附件的问答”在不兼容多模态消息格式的模型提供方上的错误处理路径。
