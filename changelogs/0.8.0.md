### v0.8.0 - 2026-03-21

## what's next?
v0.9.0 我想增加管理员账户（开发者可用），便于管理所有的团队、用户、会话和上传的文件。

## 版本概览

本版本将会话内 RAG 交互从“系统视角的文档范围配置”调整为“用户视角的附件式聊天流”：

1. 去掉顶部文档范围下拉，改为输入框左侧 `+` 轻量菜单与附件 chips。
2. 文件默认按会话级自动可用，只有在用户点选 chips 时才形成“本轮已选文件”。
3. 后端补齐文档状态、删除接口、多文档检索语义与产品化来源字段，前后端行为统一。

## 新增内容

1. 前端新增会话附件流：
   - 上传文件
   - 粘贴文本
   - 输入框上方附件 chips
   - 来源预览抽屉
2. 后端新增文档删除接口：`DELETE /api/v1/documents/{document_id}`。
3. `chat/ask` 新增 `selected_document_ids` 语义，支持按轮次限定检索文件集合。

## 主要变更

1. 交互重构
   - 移除顶部“文档范围”选择器。
   - 左侧“项目文档”收敛为“本会话文件”折叠区。
   - 导入入口改为轻量菜单，不再在主路径暴露 `source_name / content_type / max_chars / overlap / autoIndex` 这类技术参数。
2. 会话文件语义收敛
   - 文件继续归属 `conversation_id`。
   - 默认行为为自动使用本会话全部 `ready` 文件。
   - 当 `selected_document_ids` 为空或未传时，后端按当前会话可用文件自动检索。
   - 当 `selected_document_ids` 有值时，仅检索这些文件。
3. 文档生命周期显式化
   - 文档状态补齐为：`pending / chunking / indexing / ready / failed`。
   - 分块与索引流程会同步刷新文档状态，前端 chips 与侧栏列表直接显示。
4. 来源展示产品化
   - `sources` 返回补齐 `source_name + snippet + chunk_index`。
   - 前端回答下方改为“引用来源：文件名 · 第 N 段”，点击后打开文件预览抽屉。
   - 不再向用户展示 `doc_id / chunk / score` 这类底层检索信息。

## 兼容性说明

1. 为兼容已有调用链，`document_id` 语义仍保留；新前端已切换为优先使用 `selected_document_ids`。
2. 当前导入能力仍然只支持 `.txt / .md` 文本文件，尚未扩展到 PDF 解析。
