# v0.12.2 - 2026-03-27

## 版本概览

本版本统一整理 `0.12.2` 的前端与交互改动，重点是把聊天页从“功能可用”提升为更稳定、清晰、连贯的资料工作台，同时保留并扩展附件预览能力。

## 主要更新

1. 底部附件条支持双击文件项直接预览文件，单击仍用于切换本轮检索范围。
2. 侧栏文件列表与底部附件条复用同一个预览入口，文档预览行为保持一致。
3. 聊天页主界面重新整理为“资料工作台”结构，补充更清晰的顶部工作区信息、资料范围说明与场景卡入口。
4. 侧栏会话列表支持整卡点击切换，不再只能点击标题文字。
5. 会话置顶标签、选中态、附件条、输入区、消息卡片等视觉样式统一收敛到同一套绿色系设计语言。
6. 修复品牌区被滚动裁切、原生文件选择控件漏出、输入区对齐不稳、旧登录缓存导致半登录显示等前端问题。

## 交互与 UI 调整

1. 左侧品牌区从滚动区域中拆出，固定显示，避免滚动后只剩一部分可见。
2. 会话卡片整块可点击，`...` 菜单区域保持独立，避免误触。
3. 底部原生 `file input` 重新隐藏，仅保留加号菜单作为统一附件入口。
4. 加号按钮重新校正为视觉居中，输入区与发送按钮的高度、间距和对齐方式一并收紧。
5. Hero 区、状态信息和场景卡布局重做，减少冗余信息堆叠，提升首屏可读性。
6. 文档列表补充文件类型、大小、状态与失败提示，附件条帮助文案会根据 `ready / processing` 状态动态变化。
7. 发送问题时新增“处理中”占位消息，回答返回后再替换为正式结果，降低等待期的突兀感。
8. 新会话首轮提问后会自动尝试生成更有意义的会话标题。

## 兼容性说明

1. 不改动后端接口，仅整理前端结构、样式与交互逻辑。
2. 现有会话、附件选择、文件预览、删除文件、模型切换等原有能力保持可用。
3. 为避免旧浏览器缓存混用，聊天页资源版本统一继续归置为 `0.12.2`。
4. 对旧的本地登录缓存做了补齐兼容，避免出现界面局部显示未登录、局部显示已登录的状态错乱。
