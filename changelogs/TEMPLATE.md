---

填写规则：

1. 写之前至少回看 3 份历史报告：
   - 当前版本前一个版本；
   - 最近一个同模块 / 同链路版本；
   - 最近一个同类型版本（功能 / 修复 / UI / 链路）。
2. 不要只看最新一个 changelog 就开写，必须先写出“前情与承接”。
3. `版本概览 + 主要更新 + 验证结果` 建议默认保留。
4. `兼容性说明`、`涉及文件 / 已实现` 至少保留一个。
5. 问题修复型版本优先使用：`问题现象 -> 根因分析 -> 修复内容`。
6. 链路型版本优先使用：`处理链路 / 流程说明 -> API / 数据模型 / 配置变更`。
7. 交互整理型版本优先使用：`主要更新 -> 交互与 UI 调整 -> 兼容性说明`。
8. 文件路径尽量写到具体文件，不只写目录。
