# v0.14.4 - 2026-04-07

## 背景与目标

`v0.14.4` 承接了两类产品问题：

1. 首页过早暴露 `Admin`、`account_id / account_name`、模型与密钥配置，界面更像开发台而不是终端用户工作台。
2. 工作台重复进入时，前端通过重复调用 `POST /api/v1/teams` 与 `POST /api/v1/users`，再把 `409 Conflict` 当作成功继续启动，导致后端日志长期存在非绿噪音。

这个版本的目标，就是同时收束前台信息架构，并把工作台进入链路改成真正的幂等初始化流程。

## 版本概览

1. 首页主界面回到终端用户工作台心智，隐藏开发者入口。
2. “登录 / 切换账户”重构为“进入 / 切换工作台”，让文案与 namespace 语义对齐。
3. 模型、向量模型、`API Key`、`Base URL` 迁入二级设置和独立弹层。
4. 工作台初始化改为幂等 `ensure` 流程，重复进入不再制造 `409 Conflict` 日志。

## 主要更新

### 1. 主界面去开发者心智

1. 首页顶部移除 `Admin`、模型、向量模型的直接入口。
2. 主界面只保留聊天、资料增强、收藏整理等终端用户主流程。
3. `Admin` 后台继续保留在 `/web/admin.html`，但不再占据首页主信息架构。

### 2. 工作台语义替代账户语义

1. “登录 / 切换账户”改为“进入 / 切换工作台”。
2. `account_id` / `account_name` 改写为“工作台 ID / 工作台名称”。
3. 界面明确系统按工作台隔离会话、资料和偏好设置。

### 3. 高级配置迁入二级设置

1. 新增“工作台设置”入口和 `settingsModal`。
2. 默认回答模型、默认向量模型、自定义模型接入都收进二级配置面板。
3. 自定义模型与向量模型改为表单弹层填写 `provider / base_url / api_key`，不再依赖浏览器 `prompt`。

### 4. 工作台进入链路改为幂等 ensure

1. 新增 `PUT /api/v1/teams/{team_id}` 与 `PUT /api/v1/users/{user_id}`。
2. 首次创建返回 `201 Created`，重复进入同一工作台返回 `200 OK`。
3. 前端启动链路改为直接调用 `ensureTeam` / `ensureUser`，不再依赖 `"already exists"` 文案吞掉 `409 Conflict`。
4. 保留 `team_id` / `user_id` 的 64 位长度校验。
5. `ensure_user` 会阻止同一 `user_id` 跨工作台复用，避免把别的工作台用户误判为可复用对象。

## 兼容性说明

1. `Admin` 后台没有删除，仍可通过 `/web/admin.html` 进入，并继续使用 `X-Dev-Admin-Token` 鉴权。
2. 原有 `POST /api/v1/teams` 与 `POST /api/v1/users` 仍保留显式创建语义；新 `PUT` 路由用于幂等确保。
3. 本次不涉及数据库 schema 迁移，主要是前端信息架构重排与工作台初始化契约修正。
