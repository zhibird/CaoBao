# v0.14.3 - 2026-04-06

## 前情与承接
`v0.14.2` 已经把聊天消息卡片的首层动作收束为“收藏”，并明确了产品口径：

1. 用户语言应该是“先收藏”，而不是在消息出现当下就被迫区分记忆、结论、资料库。
2. `favorite / memory / conclusion / library` 这些结构化能力继续保留在系统层，但不再并列暴露在消息卡片首层。
3. 真正缺的不是能力模型，而是收藏之后的承接页和闭环动作。

因此 `v0.14.3` 解决的是上一版留下的两个关键断点：

1. 用户点击“收藏”后，前端没有收藏页承接后续整理动作。
2. “已收藏”不可再次点击取消，而且收藏后的整理动作会重复创建资产。

## 版本概览

本版本主要完成：

1. 新增收藏夹工作区，作为“先收藏，后整理”的正式承接页。
2. 将消息卡片上的收藏改为可逆 toggle，再次点击“已收藏”即可取消收藏。
3. 为收藏后的二级整理动作补上已存在状态检测，避免重复创建长期记忆、结论和资料库文档。

## 主要更新

1. 主界面新增“聊天 / 收藏夹”工作区切换，收藏夹展示当前空间下的已收藏回答。
2. 收藏夹卡片支持继续整理为“长期记忆 / 结论 / 资料库”，并保留“复制回答 / 回到对话 / 取消收藏”操作。
3. 聊天消息卡片中的“收藏”已支持再次点击取消，不再是一次性不可逆操作。
4. 收藏夹会额外拉取当前空间下已存在的 memory / conclusion / library 资产状态，已整理过的按钮会禁用并显示完成状态。
5. 新增静态前端回归测试，锁住收藏夹入口、收藏 toggle 能力和二级整理动作接线。

## 问题现象

1. 上一版用户可以收藏，但收藏之后没有前端承接页，不知道下一步去哪里整理。
2. 消息卡片里的“已收藏”只是状态展示，无法取消收藏。
3. 收藏页里的“设为长期记忆 / 沉淀为结论 / 发布到资料库”如果可重复点击，会在后端生成重复资产。

## 修复内容

### 1. 收藏夹工作区落地

1. 在主界面顶部新增“聊天 / 收藏夹”切换。
2. 收藏夹展示当前空间内的 favorites 列表，而不是做全局混合视图。
3. 收藏夹卡片按“标题 / 问题 / 回答 / 操作区”组织，承担后续整理入口。

### 2. 收藏改为真正闭环

1. 消息卡片上的“收藏”改为 toggle。
2. 若该回答已收藏，再次点击“已收藏”会直接调用删除接口并取消收藏。
3. 收藏和取消收藏后，消息卡片状态与收藏夹列表会同步刷新。

### 3. 二级整理动作补去重保护

1. 收藏夹切换进入时，会额外加载当前空间下的 memory / conclusions / library 文档状态。
2. 若某条收藏已经转为长期记忆，则“设为长期记忆”按钮会显示完成态并禁用。
3. 若某条收藏已经沉淀为结论，则“沉淀为结论”按钮会显示完成态并禁用。
4. 若某条收藏已经发布到资料库，则“发布到资料库”按钮会显示完成态并禁用。

## 交互与 UI 调整

1. `workspace` 顶部新增视图切换控件，不再让收藏能力藏在消息卡片内部。
2. 收藏夹使用独立卡片布局，弱化“配置感”，强调“先收藏，再整理”的工作流。
3. 收藏卡片中的“取消收藏”放在同一层级，避免用户只能在聊天消息里撤销。

## 验证结果
