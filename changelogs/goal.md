**目标定义（简历级）**
1. 从 Demo 升级为可上线的多会话智能知识系统。
2. 具备企业常见能力：鉴权、隔离、审计、可观测、CI/CD、性能基线。
3. 每个阶段都有量化指标，不做“功能堆砌”。

**12 周规划**
| 阶段 | 时间 | 核心目标 | 关键交付 | 验收指标 |
|---|---|---|---|---|
| Phase 2 | 第1-2周 | 数据与接口工程化 | Alembic 迁移、会话重命名、消息删除、分页查询、错误码规范 | 后端测试覆盖核心路径，API 文档完整 |
| Phase 3 | 第3-5周 | 生产级 RAG 管线 | 异步导入（队列）、分块版本化、索引重建任务、引用来源标准化 | RAG 评测集上线，Recall@5 和命中率可追踪 |
| Phase 4 | 第6-8周 | 安全与租户治理 | JWT/OIDC、RBAC、租户隔离校验、审计日志、限流 | 安全扫描无高危，越权用例全部拦截 |
| Phase 5 | 第9-10周 | 稳定性与可观测 | OpenTelemetry、Prometheus/Grafana、结构化日志、告警规则 | 可观测三件套齐全，SLO 仪表盘可用 |
| Phase 6 | 第11-12周 | 交付与简历包装 | CI/CD（灰度+回滚）、压测报告、架构图、技术复盘文档 | 能一键部署，给出性能与成本数据 |

**技术栈升级建议（企业常见）**
1. 数据库从 SQLite 迁移到 PostgreSQL（可加 pgvector）。
2. 引入 Redis + 异步任务队列处理导入/切分/索引。
3. 文件存储改为对象存储（MinIO/S3 兼容）。
4. 发布链路使用 GitHub Actions + Docker + 环境分层（dev/staging/prod）。
