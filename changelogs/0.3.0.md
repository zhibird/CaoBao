# v0.3.0 - 2026-03-18

## 版本概览

本版本围绕会话体验做了三项核心增强：

1. 支持会话重命名。
2. 支持删除单条消息。
3. 支持编辑用户消息并重新生成回复。

## 新增内容

1. 新增会话重命名接口：`PATCH /api/v1/conversations/{conversation_id}`。
2. 新增消息删除接口：`DELETE /api/v1/chat/history/{message_id}`。
3. 新增消息编辑接口：`PUT /api/v1/chat/history/{message_id}`。
4. 前端新增会话“改名”按钮，以及用户消息“编辑/删除”操作按钮。

## 主要变更

1. 会话重命名复用现有 team/user 访问校验，避免新增权限分支。
2. 消息删除按 `team_id + user_id + conversation_id` 校验归属后执行，保证会话隔离。
3. 消息编辑更新同一条历史记录（`message_id` 不变），并触发重新生成回复。
4. `action` 类型消息显式禁止编辑，避免工具调用语义被改写。
5. 前端交互简化：消息操作统一在用户消息卡片内完成，操作后自动刷新历史。

## 兼容性说明

1. 本次仅新增接口与前端交互，不涉及数据库表结构变更。
2. 既有接口行为保持兼容，原有会话与问答流程不受影响。

## 验证结果

1. 后端测试通过：`python -m pytest -q`，`24 passed in 2.02s`。
2. 前端语法检查通过：`node --check app/web/app.js`。

## 已实现

1. 会话重命名
   前端会话列表新增“改名”按钮，调用后端 PATCH /api/v1/conversations/{conversation_id}。
   代码在 app.js 和 conversation.py。

2. 消息删除
   新增 DELETE /api/v1/chat/history/{message_id}，按 team_id/user_id/conversation_id 校验后删除该轮消息记录。
   前端用户消息新增“删除”按钮。
   代码在 chat.py、chat_history_service.py、app.js、styles.css。

3. 用户消息重新编辑
   新增 PUT /api/v1/chat/history/{message_id}：编辑用户消息后，后端重新生成回复并更新同一条历史记录（message_id 不变）。
   前端用户消息新增“编辑”按钮，编辑成功后自动刷新当前会话。
   代码在 chat.py、chat.py(schema)、chat_history_service.py、app.js。
