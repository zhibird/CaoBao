# v0.17.0 - 企业工作台前端大更新

发布日期：2026-04-19

## 背景与目标

`v0.17.0` 承接 `v0.16.0` 的登录鉴权闭环，但这次整理的重点已经从“能安全进入工作台”转向“进入之后是否足够顺手、清晰、像一个成型产品”。

相对 `v0.16.0`，当前工作区这一阶段的改动范围主要集中在前端，并且不仅包含已提交内容，也覆盖了截至当前工作区仍未提交的前端增量。因此，这份日志总结的是 **`v0.16.0` tag 到当前工作区状态** 的整体变化。

这一阶段主要要解决的问题有：

1. 聊天主工作台虽然功能齐全，但视觉层次、导航结构和状态切换仍然偏“原型感”。
2. 会话、文件、设置、收藏夹等能力都已存在，但承接方式还不够统一，容易打断主聊天流。
3. Chat、Favorites、Admin 三个界面的视觉语言不够成体系，缺少统一的桌面工作台气质。
4. 收藏后的后续整理链路已经打通，但前端缺少更完整的列表-详情-继续沉淀承接面。

这个版本的目标，就是把首页工作台正式升级成一套更完整的企业级桌面工作台前端：统一主题、重做壳层、明确状态机、补齐收藏夹详情面，并让 Admin 后台也进入同一设计体系。

## 版本概览

1. 新增共享企业级主题层，Chat 与 Admin 统一接入 `theme.css`。
2. 聊天工作台从“传统侧栏页面”重构为“桌面工作台 + Rail + Dock + Recall Surface”结构。
3. 聊天启动态、首条消息进入聊天态、文件接入、设置弹层等前端状态机被重新梳理。
4. 收藏夹工作区升级为左右分栏详情视图，并支持继续沉淀为记忆、结论、资料库及反向撤销。
5. Admin 后台视觉样式跟进到同一产品家族。
6. 前端回归测试显著扩容，锁定 HTML / CSS / JS 结构与交互约束。

## 主要更新

### 1. 共享企业级主题基础落地

1. 新增 `app/web/theme.css`，把颜色、圆角、阴影、玻璃感面板、soft-grid 背景和统一动效抽成共享设计 token。
2. Chat 首页与 `Admin` 后台同时接入共享主题，不再各自维护一套基础变量。
3. 增加 `prefers-reduced-motion` 兼容，统一约束非必要动画。
4. 整体视觉风格从“功能页”升级为更稳定的桌面工作台质感：柔和背景光晕、浅玻璃面板、绿色主强调色、网格化氛围层。

### 2. 聊天工作台壳层重构为桌面工作台

1. 左侧从旧式大侧栏调整为可折叠 `workspace rail`，承担新建聊天、会话历史、工作台账户入口等核心导航。
2. 新增 `railToggleBtn` 与 `railMode` 持久化，支持展开 / 收起侧边导航状态并记住用户偏好。
3. 会话历史从独立抽屉回收到 rail 内部，当前实现以 rail 内嵌历史为主；文件与设置则保留为按需召回的叠层表面。
4. 顶部工作区切换条升级为更清晰的 `聊天 / 收藏夹 / Admin` 结构，主工作区导航层级更明确。
5. 首页启动态加入品牌条、场景卡、启动引导和更强的桌面布局骨架，让“首次进入”和“持续工作中”两个阶段有明显区分。

### 3. 聊天启动态、聊天态与资料接入逻辑重做

1. 新增 `workspaceStage`，把聊天工作台拆成 `launch` 与 `chat` 两个显式阶段。
2. 登录成功后不再直接把用户丢进旧会话，而是回到启动态，并要求先创建一轮“fresh conversation”开始新的聊天入口。
3. 首条消息发送后，界面从启动态平滑切换到聊天态，收起 Hero 区，强化消息主舞台与底部 Dock。
4. 新增 `composerContextRow`，把当前回答模式、文档数量、ready 状态、选中文档数量统一收纳成轻量 context chips。
5. 文件入口被收进 Dock 上下文区，没文件时可直接触发上传；有文件时则作为本会话文件管理入口。
6. 资料增强不再强调手动切换模式，而是改成“有 ready 文件就自动接入回答”，并支持继续缩小到指定文件范围。
7. 文件和会话列表都补了 surface status 展示，加载失败时能在对应表面直接看到错误，而不是只靠全局 toast。
8. `Escape` 与点击空白区域关闭临时表面的行为进一步补齐，文件表面与设置表面的退出逻辑更统一。

### 4. 收藏夹工作区补齐为真正的承接面

1. 收藏夹工作区从单纯列表升级为左右分栏布局：左侧是收藏列表，右侧是详情面板。
2. 当前选中的收藏会在右侧展示标题、时间、问题、回答和后续动作，不再需要只靠列表卡片承载全部信息。
3. 前端补齐收藏继续沉淀能力，支持把收藏回答继续转为长期记忆、结论或资料库文档。
4. 对已经沉淀的资产，前端也支持反向操作，包括取消长期记忆、归档结论和从资料库移除。
5. 收藏夹资产加载时会过滤已归档结论，只保留当前仍应展示的有效沉淀结果。
6. 当未登录、当前空间没有收藏、或收藏已变化时，收藏夹会有更清晰的空态与默认选中逻辑。

### 5. Admin 后台纳入同一设计体系

1. `Admin` 页面接入共享主题层，整体颜色、圆角、面板层级与主工作台统一。
2. 后台头部升级为 `admin command bar`，统计区和表格区改为更稳定的企业面板布局。
3. 文档预览弹层补齐 `Escape` 快捷关闭，后台交互细节更贴近日常操作习惯。
4. Admin 仍保持更偏结构化、密集的信息布局，但已经不再像“另一个产品”。

### 6. 前端回归测试覆盖显著扩容

1. `tests/test_web_assets.py` 新增大量针对 Chat / Favorites / Admin 的结构回归断言，锁定主题文件、工作台外壳、Dock、Rail、状态机和表面切换逻辑。
2. `tests/test_admin.py` 补充 Admin 相关交互验证。
3. `tests/test_favorites_and_conclusions.py` 补充收藏继续沉淀与撤销链路验证。
4. 这让这次前端升级不再只是视觉重排，也具备了更强的结构回归保护。

## 兼容性说明

1. 这次主要是前端工作台重构，不涉及新的数据库 schema 主路径切换。
2. `Admin` 后台仍保留在 `/web/admin.html`，只是外观和信息组织方式被统一升级。
3. 对用户来说，资料接入心智从“手动打开资料增强”进一步转向“ready 文件默认参与回答，需要时再缩小范围”。
4. 登录成功后的默认落点从“恢复到旧工作区”调整为“回到启动态并准备一轮新的聊天”，这是一次明确的交互语义变化。
5. 当前日志统计范围包含 `v0.16.0` 之后直到当前工作区的前端改动，因此其中部分内容可能尚未形成独立提交记录。
